# Security Policy

## Supported Versions
This is an active portfolio project. Security fixes target the latest `main` branch state.

## Reporting a Vulnerability
Please do not open public issues for sensitive vulnerabilities.

Instead:
1. Email: `security@placeholder.invalid`
2. Include steps to reproduce, potential impact, and any proof-of-concept details.

You can also open a private advisory if this repository is hosted on GitHub.

## Scope
Priority security areas:
- Authentication/session handling
- Supabase credentials and sync endpoints
- Data exposure via logs or debug tooling
- Dependency vulnerabilities in CI/runtime toolchain

## Response Expectations
Best-effort response timeline:
- Initial acknowledgement: within 7 days
- Triage/update: within 14 days

As this is a portfolio project, timelines may vary.
